# CLAUDE.md — 「小知識分享」資料夾守則

這個資料夾放**給欣新網電商同事看的圖文小知識**（單檔 HTML，打開就能看、能照著做）。
做這裡的任何一份檔案，都要先服從下面三條最高守則。

---

## ⭐ 最高層級守則（做每一份都先照這個）

### 1. 圖多文少、超級好理解
- **先想圖、再想字**：一個觀念先用圖示／圖卡／流程箭頭表達，文字只當補充。
- 能用一個 emoji 或圖示講清楚的，就不要寫三行字。
- 每段字數壓到最少；一句話講得完就不要兩句。
- 用**生活比喻**降低理解門檻，優先挑同事天天在做的事（例：企劃案「另存新檔 _v1 _v2 _v3」＝版本控制）。
- 一個比喻怕有人不懂，就並排放兩個不同的比喻圖卡，讓人挑看得懂的那個。

### 2. 寫給欣新網電商同事看的「0 基礎手冊」
- 讀者設定：**電商業務／企劃，完全沒有程式概念、不會打指令**。
- 角色是**和 Claude Code 合作**——「你動嘴、Claude 動手」，不是叫他自己學技術。
- 一律**繁體中文、白話**，先講「這是什麼、對我有什麼用」，再講怎麼做。
- 專有名詞（commit、push…）最多附註一次「工程師叫它 XX」，不要求讀者記，重點是他知道**要跟 Claude 說什麼**。
- 技術細節能藏就藏；讀者只需要「看得懂＋知道下一步做什麼」。

### 3. 指令（要複製的話術）一定要 100% 達到成果
- 讀者複製的是**「要對 Claude Code 說的中文句子」**，不是終端機指令。
- 每一句都必須**放明確的觸發關鍵字**，確保任何人養出來的 Claude 都會做對的事、不會誤解。
  - 例：要版本控制就一定要出現「**git**」——「幫我存起來」太模糊（可能被當成存檔案／記記憶），要寫「**幫我用 git 存一個版本**」。
- 句子要**簡潔**：能懂 ＋ 叫得動 Claude 最重要，不要多餘的字。
- 需要 Claude 自己判斷、產生的內容（例如版本說明文字）**不要叫讀者自己寫**，交給 Claude，並在旁邊小字註明。
- 每句話術都附**一鍵複製鈕**。

## 施工慣例（沿用同一套版型）
- 檔名：**首頁固定 `index.html`**（GitHub Pages 只認這個當網站首頁）；之後新增其他主題用 `0X_主題_圖文.html`，並由 `index.html` 當目錄頁連過去。
- **不要用中文檔名放到網站上**：中文會被編碼成一長串亂碼網址（`%E5%9C%96...`），同事很難貼、也容易貼壞。
- 純前端、自帶樣式、**無外部依賴**（雙擊就能開，同事不用裝任何東西）。
- 沿用第一份 `01_Git與GitHub_新手圖文.html` 的視覺風格：頂部漸層標題卡、圓角白卡片、彩色圖示、暖橘重點框、話術泡泡＋複製鈕。
- 做完在瀏覽器逐段看過畫面再交件；更新 `進度.md`。

## 驗收前自問三句
1. 有沒有「圖比字多」？哪裡還能把字換成圖？
2. 一個完全不懂程式的同事，看一遍能不能懂、知道下一步做什麼？
3. 每句要複製的話術，換任何人的 Claude 執行，會不會 100% 做到我要的結果？
