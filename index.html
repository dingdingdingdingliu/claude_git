🕰️ Git ＝ 專案的「時光機」

每存一次，就多一個「存檔點」；改壞了，隨時整個跳回之前任何一版。
🎮
像打電動存檔
🚩—🚩—🚩
過關存一次
失敗就讀取回去
📄
像企劃案另存新檔
v1—v2—v3
改壞了
打開舊的那版
↩️
Git 自動幫你做這些，還記得每版差在哪 —— 你只要說一句「幫我用 git 存一個版本」。

🧩 實際會遇到的狀況

✅
第一版功能確認 OK先存一個版本
v1
↓
🔧
做了一些優化再存一個版本
v2
↓
⚠️
測試發現原本 OK 的功能壞了優化不小心把它改壞
v2
↩️
直接退回 v1 —— 原本好的功能馬上回來，不用從頭修、也不用重做。

💬 你最常用到的三句話

💾幫我用 git 存一個版本 說明文字 Claude 會自己幫你寫
↩️幫我用 git 回到上一個版本
🔍幫我用 git 看之前存過哪些版本
💡 唯一的習慣：做完一個段落，就跟 Claude 說一句「幫我用 git 存一個版本」。存得越勤，越不怕改壞。
🚀想聽懂工程師講的 commit、push？

🔤 其實你已經在用了

初階版說的「存一個版本」＝ commit；「上傳備份」＝ push。只是換成工程師的講法。

只 commit 沒 push ＝ 東西只在你電腦裡，同事還看不到。
💡 可以先 commit 好幾次（存好幾版），做到一個段落再 push 一次就好。

💬 進階兩句話

💾幫我用 git commit 存一個版本 說明文字 Claude 會自己幫你寫
☁️幫我用 git push 上傳到 GitHub 要先 commit 過才有東西可以上傳

☁️ 那 GitHub 是什麼？ push 上去的地方

💻存在你電腦
（Git）
➜push
☁️多備一份到雲端
（GitHub）

換電腦、多人一起做才需要。要用時跟 Claude 說：「幫我用 git push 上傳到 GitHub」就好。
